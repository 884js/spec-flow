# progress.md フォーマット定義

実装進捗を管理するドキュメント。plan.md のタスク表をベースに、実行状態を追跡する。

## frontmatter

| フィールド | 必須 | 説明 |
|-----------|------|------|
| plan | ○ | plan.md への相対パス（常に `"./plan.md"`） |
| feature | ○ | kebab-case の機能名 |
| started | ○ | 開始日（YYYY-MM-DD） |
| updated | ○ | 更新日（YYYY-MM-DD） |
| mode | ○ | `single` or `multi-pr` |
| repositories | × | 対象リポジトリ一覧（YAML リスト: name, path, description） |
| docs | × | 関連ドキュメント（YAML リスト: パスや URL） |

## セクション構成

### 現在の状況
- 2-4文。完了済みの成果 + 進行中の作業 + ブロッカー（あれば）

### 次にやること
- 1-3文。再開時にそのまま実行指示として使える具体性

### タスク進捗

| # | タスク | 対象ファイル | 見積 | PR | リスク | 状態 |
|---|-------|------------|------|-----|--------|------|

**状態列の値**:
| 記号 | 意味 |
|------|------|
| `-` | 未着手 |
| `→` | 実施中 |
| `✓` | 完了 |

> タスク定義の詳細は [plan.md](./plan.md) を参照

### デリバリープラン
- single の場合: 「分割なし（1 PR）」
- multi-pr の場合: PR 一覧テーブル + Mermaid 依存関係図 + 判断根拠 + リスク軽減策

| PR | ブランチ | PR URL | 状態 |
|----|---------|--------|------|

### 作業ログ

| 日時 | 内容 |
|------|------|

## 更新ルール

- plan.md は **読み取り専用** — 一切変更しない
- タスク番号（`#` 列）は plan.md の値をそのまま使う
- 「現在の状況」「次にやること」はタスク完了・開始ごとに更新する
- frontmatter の `updated` はタスク完了ごとに更新する
