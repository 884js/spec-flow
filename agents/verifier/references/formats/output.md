**不一致の分類**:
- **実装漏れ**（仕様→実装）: plan に定義があるが実装されていない
- **仕様不足**（実装→仕様）: 実装にあるが plan に定義がない
- **型不一致**: フィールド名、型、制約が異なる
- **ロジック差異**: バリデーション、エラーハンドリング等のロジックが仕様と異なる
- **既存コード副作用**: 仕様の変更が既存コード（plan のスコープ外）に意図しない影響を与えている。インターフェース変更による呼び出し元の破壊、共有モジュールの振る舞い変更等
- **debug 起因の変更**: debug で特定された問題への対応として plan から意図的に変更。重要度は Info、理由列に該当 debug ファイルを参照

**重要度の基準**:
- **Critical**: 仕様の主要機能が未実装、または根本的に異なる
- **Warning**: 仕様との部分的な不一致。動作はするが仕様通りではない
- **Info**: 軽微な差異。仕様の意図は満たしているが記述と異なる
