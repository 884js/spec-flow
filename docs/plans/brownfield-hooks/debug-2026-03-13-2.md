# 不具合調査: スキル追跡機能が不要であることの確認

調査日: 2026-03-13

## 関連する過去の調査
- `debug-2026-03-13-1.md`: PreToolUse で Skill ツールのスキル名が取得できない問題。候補A（UserPromptSubmit パース）を実装済み

## 症状
パターン: 期待 vs 実際
- 期待: スキル追跡（Active Skill コンテキスト注入）が Claude の振る舞いを改善する
- 実際: 全6スキルの SKILL.md に `Active Skill` 出力への参照なし。注入された情報を使う指示がどこにもない

## 原因
スキル追跡機能は設計時に「Claude がスキルを忘れる」対策として導入されたが、以下の理由で不要:

1. Claude Code は SKILL.md ロード時点でスキルコンテキストを保持済み
2. 全6スキルが `Active Skill` 出力を参照していない
3. PostToolUse のフェーズ検知は STATE_FILE を参照しておらず、スキル追跡なしで独立動作する
4. 全 UserPromptSubmit で bash プロセス起動 + find 走査が発生するコスト

## 修正方針
### コード修正
- `hooks/phase-detector.sh`: UserPromptSubmit / SessionEnd セクション削除（L76-L145）。PostToolUse のみ残す
- `hooks/hooks.json`: UserPromptSubmit / SessionEnd エントリ削除。PostToolUse のみ残す
- STATE_DIR / STATE_FILE 関連のコード削除（状態ファイル管理不要）
