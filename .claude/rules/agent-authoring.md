# Agent 定義ファイル編集ルール

## Frontmatter

- **name**: エージェント名（ディレクトリ名と一致）
- **description**: エージェントの役割を簡潔に記述
- **tools**: 使用可能なツールをカンマ区切りで列挙（例: `Read, Write, Edit, Glob`）
- **model**: `opus` / `sonnet` / `haiku` のいずれか

## ディレクトリ構造

```
agents/{name}/
├── {name}.md          # エージェント定義（メイン）
└── references/        # 参照リソース
    ├── formats/       # 出力フォーマット定義
    ├── templates/     # テンプレート
    └── examples/      # 出力例
```

## 自己完結パターン

- エージェントは自身の `references/` 配下のみを参照する
- 他のエージェントの `references/` を直接参照しない
- 出力フォーマットは `references/formats/output.md`（または種別ごとのファイル）に定義する

## Model 選択基準

| model | 用途 |
|-------|------|
| **opus** | 深い分析、精密な比較、複雑な判断が必要なタスク |
| **sonnet** | パターンマッチ、テンプレートベースの生成、標準的な変換 |
| **haiku** | 軽量な CRUD 操作、単純なファイル更新、状態管理 |

## 本文の制約

- 500行以下に収める。超える場合は `references/` に分割する
- エージェントの責務を明確に記述し、スコープ外の作業を行わないよう制約を設ける
- 出力形式（何を返すか、何を返さないか）を明示する
